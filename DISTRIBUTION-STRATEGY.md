## 🔗 内容同步方案

### 方案A: GitHub 主源 + 自动分发（推荐）

**流程**:

1. **GitHub 仓库** - 主内容源
   - 保存原始 Markdown 内容
   - 版本控制
   - Issue 收集反馈

2. **自动转换脚本** - 适配各平台
   ```bash
   # 脚本位置: scripts/distribute/
   - markdown-to-zhihu.py       # 知乎格式转换
   - markdown-to-xiaohongshu.py # 小红书格式转换
   - markdown-to-wechat.py      # 公众号格式转换
   - markdown-to-juejin.py      # 掘金格式转换
   ```

3. **分发触发** - GitHub Actions
   - 监听 main 分支更新
   - 自动转换内容
   - 生成各平台版本
   - 通知人工审核

4. **人工审核** - 质量把关
   - 检查格式适配
   - 添加平台特色内容
   - 发布到平台

**优势**:
- ✅ 单一数据源，避免内容分裂
- ✅ 自动化程度高，减少人工成本
- ✅ 版本可追溯，方便回滚
- ✅ 社区贡献可直接同步到所有平台

---

### 方案B: 手动维护 + 定期同步

**流程**:

1. **GitHub 更新** - 维护主仓库
2. **定期同步** - 每周同步一次
3. **手动适配** - 为各平台手动调整
4. **发布更新** - 发布到各平台

**优势**:
- ✅ 灵活性高
- ✅ 可针对平台优化

**劣势**:
- ❌ 工作量大
- ❌ 容易不同步

## 🛠️ 实施方案（推荐方案A）

### 步骤1: 创建分发脚本目录

```bash
mkdir -p scripts/distribute
```

### 步骤2: 开发平台适配脚本

**知乎适配器** (`scripts/distribute/markdown-to-zhihu.py`):
```python
# 功能:
# 1. 转换 Markdown 为知乎格式
# 2. 保留代码块和图片
# 3. 添加知乎特有格式（如引用块、分割线）
# 4. 添加 GitHub 原文链接
```

**小红书适配器** (`scripts/distribute/markdown-to-xiaohongshu.py`):
```python
# 功能:
# 1. 提取关键步骤
# 2. 生成图文脚本
# 3. 添加小红书 emoji
# 4. 生成封面图建议
```

**公众号适配器** (`scripts/distribute/markdown-to-wechat.py`):
```python
# 功能:
# 1. 转换为公众号 Markdown
# 2. 添加引导关注
# 3. 添加原文链接
# 4. 生成排版建议
```

### 步骤3: 配置 GitHub Actions

**workflow 文件**: `.github/workflows/distribute.yml`

```yaml
name: 内容分发

on:
  push:
    branches: [ main ]

jobs:
  distribute:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: 生成知乎版本
        run: python scripts/distribute/markdown-to-zhihu.py
        
      - name: 生成小红书版本
        run: python scripts/distribute/markdown-to-xiaohongshu.py
        
      - name: 创建 Issue 通知
        uses: actions/create-issue@v1
        with:
          title: "内容已更新，请审核后发布"
          body: "各平台版本已生成，请检查 dist/ 目录"
```

### 步骤4: 创建分发输出目录

```bash
mkdir -p dist/zhihu
mkdir -p dist/xiaohongshu
mkdir -p dist/wechat
mkdir -p dist/juejin
```

## 📝 内容版本管理

### 版本策略

**主版本** (v2.0):
- GitHub 仓库中的原始内容
- 为"真实版本"
