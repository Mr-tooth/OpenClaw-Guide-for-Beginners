# 「梦境」功能使用指南

> v2026.4.9 新增核心特性：通过REM回填技术，自动整合历史记忆，提升Agent长时记忆能力

---

## 📖 什么是「梦境」功能
「梦境」是OpenClaw v2026.4.9版本新增的核心记忆增强功能，通过类似人类睡眠时的记忆整理机制，在系统空闲时自动对历史会话、日记数据进行回填整合，将零散的记忆片段整合为结构化的长期记忆，大幅提升Agent的上下文理解能力和长期记忆准确性。

---

## ✨ 核心能力
| 能力 | 说明 |
|------|------|
| **REM记忆回填** | 自动分析所有历史会话记录，提取用户偏好、重要事件、知识内容，整合进长期记忆库 |
| **记忆时间线UI** | 可视化的Web界面，查看记忆整合过程、记忆片段来源、关联关系 |
| **自动记忆去重** | 自动识别重复的记忆内容，合并相似片段，避免记忆冗余 |
| **记忆关联推理** | 自动关联相关的记忆片段，构建知识网络，提升复杂问题回答准确性 |
| **低资源占用** | 仅在系统空闲时运行，不影响正常服务响应速度 |

---

## 🚀 启用方法
「梦境」功能默认已启用，无需额外配置：
```bash
# 查看梦境功能状态
openclaw dream status

# 手动触发一次记忆整理
openclaw dream run

# 查看记忆时间线（Web UI）
# 打开浏览器访问：http://你的服务器IP:18789/dream
```

---

## ⚙️ 自定义配置
如需调整梦境功能参数，可以在`openclaw.json`中添加以下配置：
```json
{
  "features": {
    "dream": {
      "enabled": true,
      "runInterval": 3600, // 自动运行间隔，单位秒，默认1小时
      "maxMemorySize": 10000, // 最大记忆条数，默认10000
      "autoPurgeOld": true, // 自动清理30天以上的旧记忆
      "remBackfillLevel": "balanced" // 回填级别：light（轻量）/ balanced（平衡）/ full（完整）
    }
  }
}
```

## 📊 查看记忆整合结果
1. 访问Web控制台：`http://你的IP:18789`
2. 点击左侧菜单「记忆时间线」
3. 可以查看：
   - 所有整合后的记忆条目
   - 每条记忆的来源（会话ID/时间）
   - 记忆的关联关系
   - 记忆的重要性评分

## 💡 使用技巧
1. **定期导出备份**：可以通过`openclaw dream export > memory_backup.json`导出所有记忆进行备份
2. **手动标记重要记忆**：在会话中对重要内容回复「记住这个」，会被标记为高优先级记忆，不会被自动清理
3. **清理无用记忆**：如果记忆太多出现混乱，可以通过`openclaw dream purge --before 2026-01-01`清理指定日期之前的旧记忆
4. **迁移记忆**：重装系统后可以通过`openclaw dream import < memory_backup.json`导入之前的记忆

## ❓ 常见问题
### Q: 梦境功能会占用很多资源吗？
A: 不会，默认只在系统CPU使用率低于30%时运行，不会影响正常服务，10000条记忆仅占用几十MB内存。

### Q: 可以关闭梦境功能吗？
A: 可以，在配置中设置`features.dream.enabled: false`即可，但会影响长期记忆能力。

### Q: 记忆会被同步到云端吗？
A: 不会，所有记忆都仅存储在本地服务器，不会上传到任何第三方服务器，完全隐私安全。
