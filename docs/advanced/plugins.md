# 插件使用指南
OpenClaw支持丰富的插件生态系统，可扩展各种功能能力。
---
## 📦 内置插件
### 1. 文件传输插件（v2026.5.4+内置）
✅ 官方内置插件，无需额外安装，提供文件/目录操作能力。
#### 支持的工具命令
| 命令 | 功能说明 | 权限要求 | 参数说明 |
|------|---------|---------|---------|
| `file_fetch <文件路径>` | 读取指定路径文件内容 | 读权限 | `<文件路径>`：要读取的文件绝对路径或相对工作目录路径 |
| `dir_list <目录路径>` | 列出目录下的文件和子目录 | 读权限 | `<目录路径>`：要列出的目录路径，支持相对路径 |
| `dir_fetch <目录路径>` | 批量获取目录下所有文件内容 | 读权限 | `<目录路径>`：要批量读取的目录路径 |
| `file_write <文件路径> <内容>` | 写入内容到指定文件 | 写权限 | `<文件路径>`：写入目标路径<br>`<内容>`：要写入的文件内容 |
#### 使用示例
```bash
# 读取当前目录下的README.md
file_fetch README.md
# 列出/root目录下的文件
dir_list /root
# 写入内容到test.txt
file_write test.txt "Hello OpenClaw!"
```
#### 配置说明
插件默认已启用，无需额外配置。如需限制文件操作范围，可在config.json中添加：
```json
{
  "plugins": {
    "file-transfer": {
      "enabled": true,
      "allowed_paths": ["/root/workspace", "/tmp"], // 允许操作的路径白名单
      "deny_paths": ["/etc", "/root/.ssh"] // 禁止操作的路径黑名单
    }
  }
}
```
#### 安全建议
1. 生产环境建议配置路径白名单，限制文件操作范围
2. 禁止开放/root/.ssh、/etc等敏感目录的读写权限
3. 开启沙箱模式，进一步限制文件系统访问权限
