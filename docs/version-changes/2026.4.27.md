# OpenClaw 2026.4.27 版本变更说明
发布时间：2026-04-29

## 重大变更

### Codex Computer Use
- ✅ 新增 Codex Computer Use 设置
- ✅ 支持状态/安装命令
- ✅ 支持市场发现
- ✅ 支持失败关闭的 MCP 检查
- ✅ 支持 Codex 模式桌面控制

### DeepInfra 提供商
- ✅ 新增 DeepInfra 作为内置提供商
- ✅ 支持模型发现
- ✅ 支持媒体生成/编辑
- ✅ 支持 TTS 语音合成
- ✅ 支持嵌入向量
- ✅ 支持提供商自有入门引导

### 腾讯元宝/QQBot 渠道
- ✅ 新增腾讯元宝渠道支持
- ✅ 新增 QQBot 群聊支持
- ✅ 支持流式响应
- ✅ 支持媒体上传
- ✅ 支持管道重构

### 插件/模型清单优先
- ✅ 减少 Gateway 启动工作
- ✅ 使提供商行/别名/抑制更易于审计

## 功能更新

### 沙箱/Docker
- ✅ 新增 `sandbox.docker.gpus` 透传选项
- ✅ 支持 Docker 沙箱容器运行本地 GPU 工作负载

### 安全/网络
- ✅ 新增运营商管理的出站代理路由
- ✅ 支持 `proxy.enabled` + `proxy.proxyUrl` 配置

### 插件 SDK
- ✅ 暴露共享通道路由规范化
- ✅ 支持解析器驱动的目标解析
- ✅ 支持路由比较助手

### 提供商/DeepInfra
- ✅ 新增 `DEEPINFRA_API_KEY` 入门引导
- ✅ 支持动态模型发现
- ✅ 支持媒体生成

### 通道/QQBot
- ✅ 新增完整群聊支持
- ✅ 支持 C2C 流式响应
- ✅ 统一 `sendMedia` 并支持分块上传
- ✅ 支持管道重构

## 安全修复

### Gateway/会话
- ✅ 对齐 `chat.history` 和 `sessions.list` 思考默认值
- ✅ 与拥有代理解析对齐

### Telegram/Gateway
- ✅ 限制出站 Bot API 调用
- ✅ 缓存插件别名查找，防止 Gateway 楔死

### Slack/Socket 模式
- ✅ 添加 15s Slack SDK pong 超时
- ✅ 支持可配置 ping 超时

### 自动回复
- ✅ 限制运行后待处理工具结果交付排水
- ✅ 支持进度感知空闲超时

## 性能优化

### Gateway 启动
- ✅ 重用配置快照插件清单用于启动自动启用

### 插件加载
- ✅ 使用缓存的发现模式快照加载进行只读插件能力查找
- ✅ 保持快照缓存与活动 Gateway 注册表隔离

### 模型发现
- ✅ 提供商过滤的模型列表通过显式源计划路由
- ✅ 支持稳定的权限顺序

## 命令变更

### 新增命令
- `openclaw codex setup` - Codex Computer Use 设置
- `openclaw codex status` - Codex 状态检查

### 配置变更
- 新增 DeepInfra 提供商配置
- 新增腾讯元宝渠道配置
- 新增 QQBot 渠道配置
- 新增 Docker GPU 配置

## 兼容性说明

### 向后兼容
- ✅ 完全向下兼容 2026.4.x 版本配置
- ✅ DeepInfra 默认禁用，不影响现有用户
- ✅ 腾讯元宝/QQBot 可选，不影响现有工作流

### 迁移指南
- DeepInfra 提供商自动发现
- 腾讯元宝/QQBot 需要手动配置
- Docker GPU 需要手动启用

## 已知问题

### 已修复
- 修复 DeepInfra 提供商连接问题
- 修复腾讯元宝渠道认证问题
- 修复 QQBot 群聊消息问题

### 待观察
- Codex Computer Use 需要进一步优化
- 腾讯元宝集成可能需要更多测试

## 升级建议

### 推荐升级
- ✅ 建议所有用户升级到此版本
- ✅ 特别推荐需要 Codex Computer Use 的用户升级
- ✅ 特别推荐需要 DeepInfra 的用户升级

### 升级前准备
1. 备份当前配置文件
2. 备份自定义技能和插件
3. 记录当前提供商配置（如有）

### 升级步骤
```bash
# 1. 更新 OpenClaw
openclaw update

# 2. 自动修复配置
openclaw doctor --fix

# 3. 验证 DeepInfra 配置
openclaw config get deepinfra

# 4. 测试 Codex Computer Use
openclaw codex status
```
