# OpenClaw 2026.4.25 版本变更说明
发布时间：2026-04-27

## 重大变更

### TTS/语音系统全面升级
- ✅ 新增 `/tts latest` 命令，支持语音朗读最新消息
- ✅ 新增聊天范围自动 TTS 控制，支持会话级别配置
- ✅ 新增语音角色 (Personas) 功能，支持个性化语音
- ✅ 新增每代理/每账户 TTS 覆盖配置
- ✅ 新增多个 TTS 提供商支持：
  - Azure Speech (微软语音服务)
  - Xiaomi (小米语音合成)
  - Local CLI (本地命令行)
  - Inworld (AI 角色语音)
  - Volcengine (火山引擎语音)
  - ElevenLabs v3 (高级语音合成)

### OpenTelemetry 监控覆盖
- ✅ 扩展 OpenTelemetry 覆盖范围：
  - 模型调用追踪
  - Token 使用量监控
  - 工具循环追踪
  - 执行进程监控
  - 出站消息追踪
  - 上下文组装监控
  - 内存压力监控

### 浏览器自动化增强
- ✅ 新增安全标签 URL 处理
- ✅ 新增 iframe 感知角色快照
- ✅ 新增 CDP 就绪性调优
- ✅ 新增无头模式一次性启动

## 功能更新

### TTS/WhatsApp 集成
- ✅ WhatsApp 支持 `/tts latest` 语音朗读功能
- ✅ 新增重复抑制机制，避免重复朗读
- ✅ 新增会话范围自动 TTS 覆盖

### 插件系统改进
- ✅ 新增冷插件注册表，减少启动时扫描
- ✅ 新增 `openclaw plugins registry` 命令，支持显式注册表检查
- ✅ 新增 `--refresh` 修复选项

### Control UI 改进
- ✅ 新增 PWA 安装支持
- ✅ 新增 Web Push 通知功能
- ✅ 新增 Gateway 聊天推送通知

### 提供商改进
- ✅ Azure Speech 作为内置 TTS 提供商
- ✅ 支持语音资源认证
- ✅ 支持语音列表查询
- ✅ 支持 SSML 转义

## 安全修复

### 消息安全
- ✅ 恢复不安全的提供者/运行时故障后的入站消息去重
- ✅ 应用配置的编辑模式到持久化会话记录文本

### 插件安全
- ✅ 插件启动路径改进，减少扫描开销
- ✅ 插件安装元数据统一管理

## 性能优化

### 启动性能
- ✅ 冷插件注册表减少启动时扫描
- ✅ 插件启动路径优化

### 运行时性能
- ✅ 浏览器自动化性能优化
- ✅ CDP 就绪性调优减少等待时间

## 命令变更

### 新增命令
- `/tts latest` - 语音朗读最新消息
- `openclaw plugins registry` - 插件注册表检查

### 配置变更
- TTS 配置新增多个提供商选项
- 新增 OpenTelemetry 监控配置
- 新增浏览器自动化配置选项

## 兼容性说明

### 向后兼容
- ✅ 完全向下兼容 2026.4.x 版本配置
- ✅ TTS 功能默认禁用，不影响现有用户
- ✅ OpenTelemetry 默认禁用，不影响性能

### 迁移指南
- 旧版本 TTS 配置自动迁移
- 插件注册表自动初始化
- 浏览器配置自动优化

## 已知问题

### 已修复
- 修复 WhatsApp 语音朗读重复问题
- 修复浏览器自动化 iframe 处理问题
- 修复插件注册表扫描性能问题

### 待观察
- Azure Speech 语音质量需要进一步优化
- OpenTelemetry 监控可能增加少量内存占用

## 升级建议

### 推荐升级
- ✅ 建议所有用户升级到此版本
- ✅ 特别推荐需要语音功能的用户升级
- ✅ 特别推荐需要监控功能的用户升级

### 升级前准备
1. 备份当前配置文件
2. 备份自定义技能和插件
3. 记录当前 TTS 配置（如有）

### 升级步骤
```bash
# 1. 更新 OpenClaw
openclaw update

# 2. 自动修复配置
openclaw doctor --fix

# 3. 验证 TTS 配置
openclaw config get tts

# 4. 测试 TTS 功能
/tts latest
```
